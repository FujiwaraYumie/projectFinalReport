\subsection{長崎大学}
\begin{enumerate}
\item 発アプリケーションケーションのアイディア提案 
\par 長崎大としてもともと研究対象となっていたLODを用いた観光支援アプリケーションを長崎大の案として決定した． 
\item 開発アプリケーションケーションの決定 
\par 第一回合宿で各校が持ち寄ったアプリケーションケーション案をそれぞれプレゼンし，厳選した後更にメンバを混合してよりブラッシュアップした案の中から開発アプリケーションを決定した． 
\item 開発アプリケーションケーションに関する調査 
\par  このアプリケーションのターゲットや背景，開発プラットフォームについてを，このアプリケーションに関わるメンバでの話し合いによって決定した． 
\item アプリケーションケーションの機能設定 
\par 合同会議などでアイディアとして出てきた機能の中から取捨選択し，実装する機能を決定した． 
\item アプリケーションケーション名の決定 
\par 第一回合同合宿時に決定したアプリケーション名をそのまま用いることに決定した． 
\item ビジネスモデル作成技術の習得 
\par 長崎大でひとつのビジネスモデルを作成し，ブラッシュアップした． 
\item 要求定義書の作成 
\par 開発アプリケーションケーションがどのような仕様を設けるかを記したものを，2校で項目ごとに分担して合同で作成した． 
\item 要件定義書の作成 
\par アプリケーションケーションのソフトウェア要件をまとめたものを，2校で項目ごとに分担して合同で作成した． 
\item サービス仕様書の作成 
\par アプリケーションケーションがユーザ視点でどのようなサービスを提供するかを記したものを，2校で項目ごとに分担して合同で作成した． 
\item 前期提出物の作成 
\par 2校で項目ごとに分担して，2校合同で中間報告書を作成した． 
\end{enumerate}
